# Test interview corpus — Covid pandemic (synthetic, English)

This folder contains a **synthetic** corpus of 50 short interviews about
the Covid-19 pandemic experience, designed for testing and teaching
MTA's group comparison feature. Each interview is a semi-structured
exchange (Q/R format) of 7 questions on a randomized subset of topics.

## ⚠ Important: this is fake data

These interviews were generated by a Python script. They use plausible
sentences but are **not real testimonies**. Do not draw real sociological
conclusions from this dataset. Use it only to:

1. Learn how MTA works
2. Practice statistical interpretation
3. Demonstrate the pipeline in class

## Structure

- **50 interview files** named `interview_<gender>_<age>_<income>_<id>.txt`
- **One metadata CSV** `interviews_metadata.csv` listing each file with its
  three group codes — usable directly with MTA's `--groups-from csv` option

## Distribution of profiles

| Dimension | Categories | Counts |
| --- | --- | --- |
| Gender | F, M | 28 F / 22 M |
| Age band | 18-29, 30-44, 45-59, 60+ | 12 / 18 / 12 / 8 |
| Income | low, mid, high | 18 / 19 / 13 |

The distribution is **slightly unbalanced** to reflect a realistic
field survey rather than a controlled experiment.

## What differences should students expect to find?

The generator was designed to produce three levels of group differentiation,
ranging from clear to subtle. This forces students to **actually read
p-values** rather than assume every comparison is significant.

| Grouping | Expected differentiation | Why |
| --- | --- | --- |
| **Income** | **Net** | Inequalities during Covid are well documented (job loss vs. comfortable telework). Several topics should differ strongly. |
| **Age band** | **Moderate** | Different life stages produced different concerns (precarity for the young, parenting in middle-age, isolation for the elderly). Some topics will differ, others not. |
| **Gender** | **Subtle** | Differences in domestic load and remote work style are present but more delicate. Around half the topics may distinguish. |

After running MTA on this corpus, you should observe approximately:

- For **income** at α=0.05: most pairwise comparisons (NMF k=4) come back
  significant in both Welch and Mann-Whitney, and they **survive
  Benjamini-Hochberg correction**.
- For **age** at α=0.05: several comparisons are significant raw, but
  the correction reduces them considerably — a good teaching moment
  about multiple testing.
- For **gender** at α=0.05: roughly half the topics distinguish the
  groups, half don't.

The actual numbers depend on the random seed used at generation and on
the NMF / LDA model parameters you pick.

## How to use this corpus with MTA

### Quick start — Streamlit web app

1. Launch MTA via `start_MTA` and choose option `[1] Streamlit web app`.
2. On the "Load corpus" page, upload all 50 `.txt` files plus a stopwords
   file (the included `stopwords_en.txt` works fine).
3. Run NMF and LDA on the "Topic models" page (try k=4 to start).
4. On the new "Group comparison" page, either:
   - Set the position to **2** to compare by gender,
   - Set the position to **3** for age,
   - Set the position to **4** for income,
   - Or upload `interviews_metadata.csv` to compare all three at once.

### Quick start — Command line

```bash
# Compare income groups
.venv/bin/python code/MTA_v2.py \
    --corpus    examples/test_interviews_covid_en \
    --stopwords examples/stopwords_en.txt \
    --action compare-groups \
    --groups-from filenames --group-position 4 \
    --n-topics 4

# All three dimensions at once via CSV
.venv/bin/python code/MTA_v2.py \
    --corpus    examples/test_interviews_covid_en \
    --stopwords examples/stopwords_en.txt \
    --action compare-groups \
    --groups-from csv \
    --groups-csv examples/test_interviews_covid_en/interviews_metadata.csv \
    --n-topics 4
```

## Suggested classroom exercises

1. **Interpretation exercise.** For each significant topic on the
   "income" grouping, identify the most representative words
   (page 2 in Streamlit, or `nmf_top_words.csv` in batch mode) and
   propose a sociological label. Discuss whether the label makes
   sense given the income gradient.

2. **Multiple testing exercise.** Pick the "age" grouping (k=4 gives
   24 pairwise tests). Count how many tests are significant at raw
   p<0.05, then how many survive BH correction. Discuss why this matters
   for honest reporting.

3. **Effect-size vs. p-value exercise.** For one topic, compare the
   *mean* weights between two groups (effect size) and the *p-value*
   (statistical significance). Build cases where p is small but the
   difference is sociologically trivial — or the reverse.

4. **Robustness exercise.** Regenerate the corpus with a different seed
   (`python make_test_interviews.py --seed 123 --output ./test_v2_en`)
   and rerun the analysis. How stable are the conclusions across
   different random samples of the same data-generating process?

## Regenerating the corpus

The script `make_test_interviews.py` lets you create new variants:

```bash
# Default: 50 interviews with seed 42
python make_test_interviews.py

# A different sample (same generative process, different draws)
python make_test_interviews.py --seed 123

# A larger corpus for stronger statistical power
python make_test_interviews.py --seed 42 --n-interviews 200
```

A larger corpus (200+ interviews) will produce **more reliable p-values**
and will let students see that some "borderline" findings on the small
50-interview corpus become more or less robust as n grows. This is
itself a useful lesson about statistical power.

## A note on the synthetic sentences

The sentence pool was crafted to be sociologically plausible but
deliberately simplified. Real interviews would contain hesitations,
self-corrections, idiosyncratic vocabulary, narrative arcs, and
contradictions — none of which are present here. The test corpus is a
**simplified laboratory model** of what real interview data looks like
to a topic model, not a substitute for it.
